## DMIT 高防线路到底防什么、怎么防

"高防"这个词被滥用得很厉害，所以我觉得有必要说清楚 DMIT 的防御逻辑是什么。

DMIT 的高防节点（主要集中在洛杉矶 LAX 系列）提供的是 DoS 清洗防护，流量在到达你的服务器之前会经过清洗节点过滤掉攻击流量，正常业务流量继续通过。这种架构的好处是——即便你在被打，服务器本身的负载不会因为攻击流量暴增而崩掉。

香港和日本的 Pro 系列虽然主打线路质量，但也具备基础的 DDoS 防护能力，只是防御量级和洛杉矶高防系列不在同一档次。如果你的业务主要面向大陆用户、对延迟敏感，但偶尔会遇到小规模攻击，香港 Pro 或日本 Pro 是合理选择；如果你的业务经常被打、攻击量级较大，洛杉矶 Pro 高防系列才是对的方向。

## 线路质量：CN2 GIA、软银、CMI 分别适合谁

这三条线路是 DMIT 套餐里最常出现的，搞清楚区别能帮你选对套餐。

**CN2 GIA**：中国电信的顶级商用线路，去程回程都走 GIA，延迟低、丢包少，对电信用户最友好。洛杉矶 Pro 系列走的就是这条。

**软银（SoftBank）**：日本软银的国际线路，对联通用户非常友好，延迟表现稳定，日本东京节点的 Pro 系列主要依赖这条。

**CMI（China Mobile International）**：中国移动国际线路，对移动用户友好，香港节点的 Lite 系列走这条。

我自己用的是联通宽带，测试下来日本 Pro 的延迟表现让我比较满意，晚高峰也没有明显波动。如果你是电信用户，洛杉矶 Pro 的 CN2 GIA 回程会更直接。

## 高防 VPS 的典型使用场景

不是所有人都需要高防，但有几类场景几乎是刚需：

- **游戏服务器**：MC、CS 私服、FiveM 这类游戏服务器是被 DoS 的重灾区，一旦被打就直接影响玩家体验，高防是基本配置。
- **直播推流**：推流中断一次就是直接损失，稳定性要求极高。
- **有一定规模的 Web 业务**：论坛、工具站、API 服务，只要有一定曝光度就可能成为攻击目标。
- **代理和中转节点**：这类节点本身就容易被针对，高防线路能显著降低被打离线的频率。

如果你只是跑个人博客或者轻量级应用，Lite 系列完全够用，没必要为高防多付钱。

### DMIT 支持哪些付款方式？

DMIT 支持 PayPal、信用卡以及加密货币支付，对于国内用户来说支付渠道比较灵活，不需要额外折腾。

### 被攻击超出防御上限会怎样？

超出防御量级后，通常的处理方式是临时封禁该 IP（黑洞路由），等攻击停止后自动或手动解封。这是行业通行做法，DMIT 也不例外。所以如果你的业务经常遭遇超大规模攻击，需要选择防御量级更高的套餐。

### DMIT 的VPS 支持自定义 ISO 吗？

部分套餐支持自定义 ISO 安装，具体支持情况以套餐说明页为准。常见的 Linux 发行版（Debian、Ubuntu、CentOS）都在默认可选系统列表里。

### 香港节点和洛杉矶节点哪个延迟更低？
